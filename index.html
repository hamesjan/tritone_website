ECE 196: Audio To Color Array Conversion

James Han, Uriberto Lopez, Alexis Pascual

Problem Definition

Concerts only using auditory information aren’t accessible to the hearing impaired. Relying on an interpreter or feeling the vibrations through the ground becomes impossible for some settings.

System Diagram

Our System Diagram begins by receiving audio input from the various microphones throughout the device and sends it to the teensy audio shield. The audio shield process these signals and turns them from analog to digital for the Teensy 3.6 board to process. Here, the signal is recorded in chunks. The data captured by the Teensy is given to the ESP32 where a machine learning algorithm will be applied to the data collected. We take 11 inputs; 10 dominant frequencies taken from each seconf for the last 10 seconds and the current dominant frequency and pass it through a two hidden layer, 16 parameter feed forward network to output 3 values: r,g,b. The model training was accomplished by collecting data by James and self-labeling.

Workshop: Building the Project

Step 1: Gather Materials

For this project, you will need the following materials:

  • Teensy3.6 and Audioshield
  • ESP32S3 DevKit
  • Female to Male header wires
  • Male to male wires
  • USB-C Wire
  • Micro USB Wire
  • Adafruit 64x64 RGB matrix
  • 5V 4A Switching Power Supply
  • DC Power adapter

Step 2: Wire up the hardware

We used the following wirings:

  • Format: HUB75E -> ESP GPIO
  • R1_PIN -> 46
  • G1_PIN -> 45
  • B1_Pin -> 42
  • R2_PIN -> 41
  • G2_PIN -> 40
  • B2_Pin -> 39
  • A_PIN -> 38
  • B_PIN -> 37
  • C_Pin -> 36
  • D_PIN -> 35
  • E_PIN -> 48
  • LAT_Pin -> 34
  • OE_PIN -> 33
  • CLK_PIN -> 47

Then, connect Teensy 3.3V to ESP32 3.3V and make sure to connect all grounds.

Connect Teensy pin 0 to pin 17 (TX) on ESP and Teensy pin 1 to pin 18 on ESP(RX). This is to setup UART Communication.

Connect the power to the RGB Matrix.

Connect a microphone's output to ESP32 Pin 1 and connect ground and power togther.

Step 3: Code up the software

  1. We used Adafruit's Protomatter library to program the RGB matrix and PyTorch to train the Feed forward network.
  2. After acquiring data and training a small neural network, export the network parameters and the scaling parameters for normalization and hard code the forward passes for inferencing on the ESP.
  3. We render the spectrogram data on the ESP32 from the direct microhpone input and collect frequency data transmitted over from Teensy.
  4. We collect the last 10 dominant frequencies over the last 10 seconds and keep them stored in an array. Then, every second with the new frequency, we use that as an input for our Feed Forward Network inferencing.
